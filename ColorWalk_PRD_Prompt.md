# Color Walk App — 开发 Prompt（可直接粘贴给 AI）

---

## 一、产品定义

你正在开发一款名为 **Color Walk** 的 iOS 原生应用。

**一句话描述：**
Color Walk 是一款静默的现实色彩收集游戏。用户在出发前获得一个系统生成的主题色带作为目标，带着它去散步，用相机在现实世界中寻找和采集颜色，结束后生成一张私人色卡存入档案。没有社交，没有点赞，只有一个人和颜色。

---

## 二、核心用户画像

- 年龄 20–35 岁，有审美意识
- 用过 Pinterest、VSCO、小红书
- 喜欢安静的个人记录类 App（日记、Notion）
- 设计师 / 艺术生 / 情绪敏感型用户
- 不需要社交压力，需要一个安静的容器来盛放感受

## 四、功能模块详细说明

### 4.1 出发前：主题色生成

**逻辑：**
- 系统根据当前季节自动偏向生成主题色带：
  - 春：粉藕、嫩绿、浅蓝等清浅色
  - 夏：饱和青绿、明黄、珊瑚色
  - 秋：赭红、烟褐、稻黄、橄榄绿
  - 冬：冷灰、雾蓝、霜白、深靛
- 色带为**跨色相渐变**（非单一色相由深到浅），例如：秋季可从「赭石红」渐变经「烟褐」到「稻草黄」
- 每次生成结果随机，同一季节内有多种可能的色带组合

**用户操作：**
- 系统生成后展示色带预览
- 提供「重新生成」按钮（可多次随机）
- 提供「自定义」入口：用户可手动选择渐变起点色和终点色，系统自动生成中间过渡

**展示形式：**
- 横向渐变色带，宽度占满屏幕，高度约 80pt
- 色带下方显示两端颜色的中文色名（如「砖红」→「浅杏」）

---

### 4.2 出发前：采集模式选择

两种模式，用户每次 walk 前选择一种：

**模式 A：传统 Color Walk（单色模式）**
- 整次 walk 只允许最终锁定一个颜色
- 过程中可以取色预览，但只有主动点击「锁定」才算采集
- 锁定后不可更改（增加仪式感和决策感）
- 结束时生成极简单色卡

**模式 B：自由采集（多色模式）**
- 全程自由采集，无数量上限
- 每次点击即采集，可删除已采集的颜色
- 结束时生成调色盘色卡

---

### 4.3 出发前：严格程度设置

三档，用户每次 walk 前选择（可记住上次偏好）：

| 档位 | 名称 | 含义 |
|------|------|------|
| 宽松 | 氛围漫游 | 主题色仅作为出发氛围参考，采集无限制，无匹配提示 |
| 中等 | 色彩猎人 | 采集时给出色相接近度的轻量提示，不强制 |
| 严格 | 精准采集 | 采集时实时显示当前色与目标色带的量化匹配度（0–100%），单色模式下锁定后显示最终得分 |

**单色模式 × 严格程度联动：**
- 氛围漫游 + 单色：随便找一个你觉得对的颜色锁定即可
- 精准采集 + 单色：找到匹配度最高的那个颜色，这是真正的挑战

---

### 4.4 采集中：相机取色

- 打开相机实时预览画面
- 屏幕中央有一个取色准星（圆形，直径约 20pt）
- 准星所在位置实时提取颜色，显示：
  - 颜色预览色块
  - HEX 值（如 #C4855A）
  - RGB 值（如 R196 G133 B90）
  - 中文色名（如「赭石」）— 使用预置的中国传统色名库匹配最近色
- **单色模式**：底部有「锁定此色」按钮，点击后锁定，进入结束页
- **自由模式**：底部有「采集」按钮，点击后将当前色加入色板，可继续采集；右上角显示已采集数量；手动点击「完成」结束

---

### 4.5 采集中：匹配反馈

根据严格程度设置决定反馈形式：

**氛围漫游：**
- 无任何匹配提示，只显示取色结果
- 主题色带以小图形式常驻屏幕角落（提醒氛围，不评判）

**色彩猎人：**
- 准星下方显示一个色相接近度指示条（暖色→冷色的光谱条，标注当前色位置与目标区间）
- 不显示数字，只显示位置关系

**精准采集：**
- 实时显示当前色与目标色带的匹配度百分比（0–100%）
- 匹配度 ≥ 80% 时准星边框变为目标色，给予视觉正反馈
- 单色模式锁定后显示最终匹配得分，并用一句话描述（如「完美命中」/ 「略有偏差」/ 「意外之色」）

---

### 4.6 结束：色卡生成

**单色模式生成物：**
- 极简单色卡，包含：
  - 大面积目标色带（上半部分）
  - 命中色（下半部分，占比约 40%）
  - 日期 + 时间 + 地点名称（小字，右下角）
  - 严格模式下显示匹配得分
- 风格：大留白，无多余装饰

**自由模式生成物：**
- 调色盘色卡，包含：
  - 顶部：主题色带（细条）
  - 主体：采集的所有颜色按色相排列的色块
  - 底部：日期 + 地点
- 色块可为方形或不规则拼贴风格（参考设计规范）

**操作：**
- 「保存到相册」按钮
- 「存入档案」按钮（保存到 App 内档案，不互斥）
- 「再走一次」（返回首页重新开始）

---

### 4.7 档案页

- 时间线列表，每条记录显示：
  - 主题色带缩略图
  - 采集结果缩略图（单色：命中色色块；多色：调色盘缩略）
  - 模式标签（单色 / 多色）+ 严格程度标签
  - 日期
- 点击进入该次 walk 的完整色卡详情
- 支持按「单色 / 多色」筛选

## 五、设计语言规范（必须严格遵守）

### 整体风格
- **东方禅意 / wabi-sabi**，不是科技感 App，不是相机 App
- 关键词：宣纸、留白、墨迹、竹简、枯山水
- 有意留出大量负空间，不追求信息密度

### 色彩
- 背景底色：暖白 / 宣纸色，推荐范围 `#F5F0E8` ~ `#FAF7F2`
- 文字主色：深墨色，推荐 `#1A1714`
- 辅助色：只用低饱和度的灰褐色系
- **禁止使用**：纯白 `#FFFFFF` 背景、科技蓝、渐变彩色 UI 元素（渐变只用于主题色带本身）

### 字体
- 标题 / 色名：衬线体（推荐 Noto Serif SC 或 Source Han Serif）
- 数值 / HEX：等宽字体（推荐 SF Mono 或 JetBrains Mono）
- 正文 / 按钮：细圆体（推荐 PingFang SC Light）
- **禁止使用**：SF Pro Display 粗体大标题风格、科技感无衬线堆叠

### 圆角
- 柔和：主要容器 `cornerRadius: 16`，色块 `cornerRadius: 8`
- **禁止**：过度圆润（像消消乐）或完全直角（像表格）

### 动效
- 慢、柔、克制：所有动画时长 ≥ 0.4s，使用 `easeInOut`
- 色块出现：淡入（opacity 0→1），不要弹跳
- 取色准星：颜色过渡平滑，不闪烁
- **禁止**：bounce 弹簧动画、快速 slide、任何"活泼"感动效

### 禁止出现的 UI 元素
- 相机 App 的网格线、快门按钮圆形设计
- Instagram / VSCO 风格滤镜选择器
- 霓虹色、大面积彩色背景
- 点赞、评论、分享等社交元素
- 任何游戏化的积分、徽章、排行榜

## 六、技术栈

- **平台**：iOS 原生，最低支持 iOS 16
- **语言**：Swift + SwiftUI
- **相机**：AVFoundation（实时取色用 AVCaptureVideoDataOutput 逐帧采样）
- **颜色处理**：Core Image 或自定义 UIColor 扩展，支持 HEX / RGB / HSB 互转
- **中文色名库**：本地 JSON 文件，存储约 400 个中国传统色名及对应 HEX 值，匹配算法用 Lab 色彩空间最近邻
- **本地存储**：SwiftData（iOS 17+）或 CoreData
- **图片导出**：UIGraphicsImageRenderer 生成色卡图片，写入相册用 PHPhotoLibrary
- **无后端**：所有数据本地存储，不需要网络请求

## 七、数据模型（供参考）

```swift
// 一次 Walk 记录
struct WalkRecord {
    var id: UUID
    var date: Date
    var location: String?          // 地点名称（可选，用 CLGeocoder 反地理编码）
    var mode: WalkMode             // .single / .free
    var strictLevel: StrictLevel   // .ambient / .hunter / .precise
    var themeGradient: ColorGradient  // 主题色带（起点色 + 终点色）
    var collectedColors: [CapturedColor]  // 采集的颜色（单色模式只有1个）
    var matchScore: Int?           // 匹配得分（仅精准模式）
}

struct ColorGradient {
    var startColor: CapturedColor
    var endColor: CapturedColor
}

struct CapturedColor {
    var hex: String
    var rgb: (r: Int, g: Int, b: Int)
    var chineseName: String
}

enum WalkMode { case single, free }
enum StrictLevel { case ambient, hunter, precise }
```
